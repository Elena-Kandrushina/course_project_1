# Проект 1. Приложение для анализа банковских операций

## Описание:

Приложение для анализа транзакций, которые находятся в Excel-файле.
Приложение будет генерировать JSON-данные для веб-страниц, формировать отчеты,
а также предоставлять другие сервисы

## Установка:

1. Клонируйте репозиторий:
```
git@github.com:Elena-Kandrushina/course_project_1.git
```

2. Установите зависимости:
```
poetry install
```

## Использование:

В модуле utils реализованы следующие функции:
1. time_greeting_func, которая возвращает приветствие в формате
 'Доброе утро'/'Добрый день'/'Добрый вечер'/
    'Доброй ночи' в зависимости от текущего времени
2. get_datetime, которая принимает строку с датой и возвращает список дат,
 первая соответствует началу периода, а вторая дате, переданной в функцию"""
3. get_slice_by_period, которая принимает путь к файлу operations.xlsx и временной отрезок, возвращает
    таблицу соответствующую выбранному периоду
4. get_number_card_total_spent_cashback, которая принимает таблицу с выборкой по временному периоду и возвращает
   список словарей с ключами 'last_digits', 'total_spent', 'cashback    

5. get_top_n_transactions, которая принимает таблицу с выборкой по временному периоду и число, определяющее
     количество словарей в списке (топ транзакций по сумме платежа) и возвращает список
    словарей с обозначенным количеством транзакций

6. get_currency_rates, которая принимает путь к файлу user_settings.json и возвращает
    курс валют
7. get_stock_prices, которая принимает путь к файлу user_settings.json и возвращает
    стоимость акций из S&P500 

8. get_path_return_dataframe, которая принимает путь к файлу operations.xlsx
    и возвращает DataFrame со всеми транзакциями
В модуле services реализованы следующие функции:
1. search_by_string, которая принимает таблицу с данными и строку/слово для поиска и возвращает JSON-ответ со всеми
    транзакциями, содержащими запрос в описании или категории
2. search_phone_number, которая принимает таблицу с данными и возвращает JSON-ответ со всеми транзакциями,
    содержащими в описании мобильные номера 
В модуле repotrs реализованы следующие функции:
1. report_decorator - это декоратор, котрый записывает в файл результат, который возвращает функция, формирующая отчет
2. spending_by_category, которая возвращает траты по заданной категории за последние 3 месяца
    от заданной даты или текущей даты 
В модуле views реализована главная функция-конструктор, которая собирает в себе функции из
utils и служит для вывода на страницу "Главная"
Модуль main запускает проект

## Тестирование:
# Установка через Poetry
poetry add --group dev pytest

В pytest для анализа покрытия кода надо поставить библиотеку 
pytest-cov:

Через poetry с добавлением в отдельную группу

poetry add --group dev pytest-cov

Чтобы запустить тесты с оценкой покрытия, можно воспользоваться следующими командами:

pytest --cov  — при активированном виртуальном окружении.

poetry run pytest --cov  — через poetry.

pytest --cov=src --cov-report=html — чтобы сгенерировать отчет о покрытии в HTML-формате, где 
src — пакет c модулями, которые тестируем. Отчет будет сгенерирован в папке 
htmlcov и храниться в файле с названием index.html.

В тестировании применены фикстуры для создания необходимых входных данных для тестов и параметризация для
обеспечения тестирования функциональности с различными входными данными.
Так же применены Mock и patch.
